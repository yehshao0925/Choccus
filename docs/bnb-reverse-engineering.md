### 1.1 爆炸命中 66% 重疊（重要手感點）

- 角色有矩形碰撞框；系統檢查水柱是否覆蓋 **≥66%** 才判定被困。<66% 即使在爆風範圍內也不會被困 → 這就是高手「**卡縫躲爆**」（In-Between／韓文 끼기）的原理。
- **Choccus 現況落差**：`client/src/sim/Sim.ts:210` 用的是「角色**中心點**所在那一格是火焰格 → 直接被困」的**整格二元判定**，沒有 66% 重疊。所以卡縫躲爆在 Choccus 不存在。
- 要不要補：是「想要高手操作天花板」才做的 scoped feature，需同步改 sim 命中 + AI danger map（CLAUDE.md 鐵則：AI 爆炸射線須逐字對齊 `Explosion.ts`）+ 重 pin golden。本 session 已決議**先不動**。

### 1.2 角色移速：base 普遍＝5、差在「極速上限」、滾輪鞋每顆 +1

- 速度**不是單一全域值**：每角色不同、且**初期 ≠ 極速**。但**起手 base 幾乎都＝5**（5/7、5/8、5/9 起手都 5）——**差異在極速上限**，不在起手。
- **加速道具＝滾輪鞋（롤러스케이트），每顆 +1 階**（已驗證）；紅惡魔／冰鞋＝瞬間拉滿到該角色上限。（坑：另有「鞋子 신발」是「能踢水球」**不是**加速，別混。）
- 「**吃 2 顆鞋就滿速**」＝**手遊版 Bazzi 5/7**（5→6→7）。台版 PC（TF）刻度不同 → 囡囡 5/8＝**3 顆**滿、睏寶 5/9＝**4 顆**滿。**「幾顆滿」因版本而異**，別當權威（呼應 §四：PC 台服略異）。

## 四、尚未測得的數值（與原因）

下列 **kinetic（動態手感）數值本 session 測不到**，皆因 YouTube 影片是錯的量測儀器：

| 想要的數值 | 狀態 |
| --- | --- |
| 基礎移速（格/秒實值） | ❌ |
| 加速度曲線（是否瞬間到速） | ❌ |
| **轉角輔助容差**（偏多少還能滑過去） | ❌ |
| 輸入緩衝（早按是否記住） | ❌ |
| 引信時間（放彈→爆炸） | ❌ |
| 火焰持續 / 十字延伸速度 | ❌ |
| **推軟磚的時間／速度**（村10、海盜14） | ❌（2026-06-24 三法皆敗，見下） |

**推軟磚特別記錄（2026-06-24）**：試圖從村10影片（`J3n5I8hoVpQ` 1080p60）量「推一格的時間與滑速」。把整盤量化成 15×13 的 96×72 格、逐 tile 追佔用狀態，跑了三層偵測：①floor→brick「磚出現」→ 被爆炸藍水柱淹沒（藍冰磚與藍爆炸同色，classify 無法分）；②加「出現後須維持 2.5s」濾爆炸 → 改被道具/動畫閃爍灌爆（同一 tile 一場狂跳 11 次）；③用物理定義「相鄰格同時一失一得」抓真推 → 被**爆炸波前沿沿路掃格**淹沒（波前一格一格移＝和推磚一模一樣的「鄰格失+得」簽章，還會出現 `(11,10)→(10,10)` 隔秒又 `(10,10)→(11,10)` 的自相矛盾對）。**結論：color/occupancy 全系列無解**，要確認真推必須驗證磚的 sprite「保留貼圖平移一格」＝ template/光流追蹤，而本footage 連這也糊（同 §四主因）。

**失敗原因（測過 3 支影片、6 種 CV 方法後確認，一致）：**
1. **可破壞地形** → 背景相減失效（炸掉的格永久異於背景，前景框變巨塊）
2. **高手/密集場無乾淨直走** → 追蹤抓不到等速段
3. **片頭字卡/重播** → 框不固定，中位數背景糊掉
4. **小視窗錄製** → 角色塊太小、碎裂
5. **爆炸連鎖重疊** → 火焰脈衝糊在一起，單顆持續切不乾淨
6. **影片本質看不到輸入** → 轉角容差／緩衝原理上無解
7. **per-character 速度** → 根本沒有單一「基礎速」可量

## 五、唯一可靠路徑：受控側錄

kinetic 數值要量準，必須**自己在遊戲裡錄受控片段**（受控＝無破壞、單角色、已知輸入），再用同一套 CV 量。建議錄製清單（標明 fps）：

1. **空巷直走 3~5 格**（全程不放彈）→ 數幀算基礎移速（綁定角色）＋驗證格距
2. **逼近轉角、不同偏移按轉向**（偏一點 / 偏多一點各數次）→ 量「最大還能滑過去的偏移」＝轉角容差
3. **走到底前提早按反向** → 看輸入緩衝是否記住
4. **單顆水球放置→爆炸**（空地、無連鎖）→ 引信時間＋火焰持續＋延伸速度

## 附錄：量測工具鏈（供下次續作）

- 下載：`yt-dlp -f <fmt> -o out.mp4 <url>`；查格式 `yt-dlp -F <url>`；字幕 `--write-auto-subs`
- 抽幀：`ffmpeg -ss <t> -i in.mp4 -frames:v 1 out.png`；蒙太奇 `-vf "fps=1,scale=...,tile=AxB"`（注意此 ffmpeg build 無 freetype，不能 drawtext）
- 分析 venv：`/tmp/anlz`（numpy + pillow + **opencv-contrib**-python-headless；headless 版無 CSRT tracker，要 contrib）
- 格距偵測：median 背景 → Sobel 邊緣 column/row profile → autocorrelation 取週期（**textured 地板會帶歪，需肉眼覆核**）
